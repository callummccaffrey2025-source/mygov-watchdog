# Verity Autopilot Task Queue

## How this works
Claude Code reads this file, picks the highest-priority incomplete task, implements it, verifies with `npx tsc --noEmit`, updates the status, logs the result, and moves to the next task.

## Rules
1. ALWAYS read CLAUDE.md and DESIGN.md before starting any task
2. ALL visual changes must use inline styles, NEVER StyleSheet.create
3. Every task must end with `npx tsc --noEmit` — zero errors or revert
4. One task at a time. Finish completely before starting the next.
5. If a task is unclear or risky, mark it SKIPPED with a reason and move on
6. Never modify AUTOPILOT_TASKS.md structure — only update Status and Log fields
7. Maximum 3 consecutive tasks per session to avoid context degradation
