## Resolved: politicians vs members

**Problem:** `politicians` (226 rows) and `members` (225 rows) contained the same people with different schemas. 224 name overlaps. The `politicians` table used denormalized text fields for party/electorate; `members` used proper FKs. All app code used `members` exclusively.

**Resolution:** Dropped `public.politicians`. Backup at `archived.politicians`.

| Action | Detail |
|--------|--------|
| Backup | `CREATE TABLE archived.politicians AS SELECT * FROM public.politicians` |
| Drop | `DROP TABLE public.politicians CASCADE` |
| FK cleanup | `ALTER TABLE user_notifications DROP CONSTRAINT user_notifications_politician_id_fkey` |
